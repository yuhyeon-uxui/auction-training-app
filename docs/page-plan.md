# 페이지 구조 설계서 (Page Plan)

본 문서는 '부동산 경매 가상 입찰 훈련소' 앱의 화면 라우팅 구조와 각 페이지별 주요 컴포넌트 구성을 정의합니다.

## 1. 글로벌 라우팅 트리 (Routing Tree)

앱은 React Router v6를 기반으로 동작하며, 로그인 상태에 따라 분기됩니다.

```text
App (최상위 컨테이너)
 ├── 미인증 상태 (Session == null)
 │    └── /login (Login.jsx): 로그인 및 회원가입, 구글 OAuth
 │
 └── 인증 상태 (Session != null)
      ├── 공통 UI
      │    ├── Sidebar.jsx (좌측 네비게이션)
      │    └── GlossaryPopup.jsx (전역 용어사전 팝업)
      │
      ├── / (Home.jsx) : 홈 대시보드
      ├── /study (Study.jsx) : 개념 배움터
      ├── /bid (Bidding.jsx) : 가상 입찰 훈련소
      └── /profile (Profile.jsx) : 프로필 및 성장 통계
```

## 2. 페이지별 컴포넌트 상세 구조

### 2.1. 로그인 페이지 (`Login.jsx`)
- **목적**: 사용자의 진입점 및 세션 생성
- **주요 UI 요소**:
  - 이메일/비밀번호 입력 폼
  - [로그인], [새 계정 만들기], [비밀번호 찾기] 버튼
  - [Google로 계속하기] 소셜 로그인 버튼
- **연동 데이터**: `supabase.auth`

### 2.2. 메인 대시보드 (`Home.jsx`)
- **목적**: 현재 학습 상태 요약 및 다음 행동 유도
- **주요 UI 요소**:
  - `DashboardCard.jsx` (재사용 컴포넌트): 나의 레벨, 연속 학습일, 입찰 참여 횟수 표시
  - 오늘 할 것 (Study, Bidding 페이지로 이동하는 퀵 링크 카드)
  - 최근 입찰 기록 리스트 (최대 3개 표시)
- **연동 데이터**: `profiles` 테이블 (유저 정보), `bid_history` 테이블 (최근 입찰)

### 2.3. 개념 배움터 (`Study.jsx`)
- **목적**: 경매 핵심 개념 텍스트 학습 및 미니 퀴즈
- **주요 UI 요소**:
  - 개념 설명 텍스트 블록 (용어사전 밑줄 하이라이트 적용)
  - 객관식 퀴즈 버튼 컨테이너 (정답/오답 클릭 시 즉각적인 색상 피드백)
  - 해설 창 (오답 시 재도전 기능 제공)
- **연동 데이터**: `quiz_attempts` 테이블 (퀴즈 결과 기록), `profiles` 테이블 (레벨업 반영)

### 2.4. 가상 입찰 훈련소 (`Bidding.jsx`)
- **목적**: 매물 분석 및 실전 모의 입찰 경험 제공
- **주요 UI 요소**:
  - 매물 카드 그리드 (감정가, 최저입찰가, 등기부 현황 요약)
  - 입찰가 입력 필드 (`input type="number"`)
  - [입찰하기] 버튼
  - 결과 확인 모달창 (낙찰/패찰 시각적 피드백, 수익률 계산, 오답 노트 해설 제공)
- **연동 데이터**: `properties` 테이블 (가상 매물 로드), `bid_history` 테이블 (입찰 결과 및 오차율 계산 기록)

### 2.5. 프로필 통계 (`Profile.jsx`)
- **목적**: 유저의 성장 과정을 시각적으로 증명 (게이미피케이션)
- **주요 UI 요소**:
  - 유저 레벨 및 등급 아이콘 (상단 요약)
  - 4종 차트 영역 (`Recharts` 라이브러리 사용)
    - `WeeklyStudyChart.jsx`: 주간 학습량 바 차트
    - `BidAccuracyChart.jsx`: 최근 10회 입찰 정확도 바 차트
    - `WinRateChart.jsx`: 누적 낙찰 성공률 라인 차트
    - `LevelHistoryChart.jsx`: 레벨업 타임라인 히스토리
  - 전체 입찰 기록 테이블
- **연동 데이터**: `profiles`, `quiz_attempts`, `bid_history` 테이블 데이터 종합 분석

## 3. 공통 UI 컴포넌트 디자인 시스템 (Design System)
- **카드 UI (`.card-style`)**: 배경색, 다크 테두리, 호버(Hover) 시 살짝 떠오르는 애니메이션이 통일된 Tailwind 커스텀 클래스를 사용하여 앱 전체의 일관성 유지
- **기본 버튼 (`.btn-primary`)**: 블루 포인트 컬러(`#3B82F6`)와 네온빛 그림자 효과가 일관되게 적용된 주 사용 버튼
- **용어 팝업 툴팁 (`.glossary-term`)**: 텍스트에 해당 클래스만 붙이면, 전역 컴포넌트인 `GlossaryPopup.jsx`가 이를 자동으로 인식하여 마우스 오버 시 용어 설명 팝업을 띄워주는 모듈화 설계
