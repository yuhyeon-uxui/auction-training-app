# 아키텍처 및 구현 설계서 (Implementation Plan)

본 문서는 '부동산 경매 가상 입찰 훈련소' 프로젝트의 시스템 아키텍처, 기술 스택, 구조 설계 및 벤치마킹 사례를 정의하는 테크니컬 스펙(Technical Specification) 문서입니다.

## 1. 기술 스택 및 도입 목적 (Tech Stack & Justification)

| 기술 | 도입 목적 (어떤 부분을 위해 있는가?) | 학습/구현 난이도 |
|------|--------------------------------|------------------|
| **React 18** | UI를 재사용 가능한 컴포넌트 단위로 조립하고, 상태 변화에 따라 화면을 즉시 업데이트(SPA)하기 위함입니다. | 🟠 중간 |
| **Vite** | 기존 Webpack보다 압도적으로 빠른 로컬 서버 실행 및 빌드 속도를 제공하여 개발 생산성을 높입니다. | 🟢 쉬움 |
| **React Router v6** | 페이지를 새로고침하지 않고 홈, 개념공부, 가상입찰, 프로필 화면 간의 부드러운 화면 전환을 담당합니다. | 🟢 쉬움 |
| **Tailwind CSS** | 별도의 CSS 파일 없이 HTML 클래스 이름만으로 빠르게 다크모드 및 반응형 SaaS UI를 스타일링합니다. | 🟠 중간 |
| **Recharts** | 학습량, 입찰 정확도 등의 복잡한 유저 데이터를 시각적으로 아름답고 이해하기 쉬운 그래프로 렌더링합니다. | 🟢 쉬움 |
| **Supabase** | 별도의 백엔드(서버) 개발 없이, 유저 인증(구글 로그인)과 실시간 PostgreSQL 데이터베이스를 즉시 제공합니다. | 🔴 다소 높음 |

## 2. 전반적인 아키텍처 및 구조 설계 (Architecture Design)

빅테크 및 모던 프론트엔드 진영에서 널리 사용되는 **'컴포넌트 및 관심사 분리(Separation of Concerns)'** 표준 방식을 채택했습니다.

- **`src/pages/` (뷰 계층)**: 각 라우트(URL)에 매칭되는 거대한 페이지 단위 컴포넌트를 위치시킵니다. (예: `Home.jsx`, `Bidding.jsx`)
- **`src/components/` (재사용 컴포넌트 계층)**: 페이지 내부에서 반복적으로 사용되는 UI 조각들을 분리합니다. (예: `DashboardCard.jsx`, `GlossaryPopup.jsx`)
- **`src/lib/` (유틸리티/서비스 계층)**: 외부 API(DB)와의 통신 로직을 모아둡니다. (`supabaseClient.js`)
- **상태 관리**: 복잡한 전역 상태관리 라이브러리(Redux 등) 대신, React의 기본 `useState`, `useEffect`를 활용하여 가볍게 설계했습니다. 세션(Session) 상태만 `App.jsx` 최상단에서 관리하여 하위로 내려주는 Top-Down 방식을 적용했습니다.

## 3. 유사 서비스 레퍼런스 (References & Benchmarks)

본 서비스는 딱딱한 금융 앱이 아닌, **'게이미피케이션(Gamification)'** 요소가 가미된 에듀테크/SaaS 디자인을 벤치마킹했습니다.

1. **듀오링고 (Duolingo)**:
   - *차용 포인트*: 연속 학습 스트릭(Streak) 시스템, 미니 퀴즈 형태의 개념 학습, 정답/오답 즉각 피드백 (시각적 + 컬러 활용).
2. **토스 (Toss) / 업비트**:
   - *차용 포인트*: 다크모드 기반의 네온/블루 포인트 컬러, 텍스트를 최소화하고 숫자를 크게 보여주는 직관적인 대시보드 카드 UI.
3. **일반적인 SaaS Admin Dashboard (Vercel, Supabase 콘솔)**:
   - *차용 포인트*: 좌측 고정 사이드바 네비게이션, 넓은 중앙 콘텐츠 영역, 데이터 시각화를 위한 그리드 차트 배치.

## 4. 보안 및 데이터 흐름 (Data Flow)

1. **클라이언트 → Supabase**: 모든 DB 접근은 클라이언트에서 직접 수행하되, Supabase의 **RLS(Row Level Security)** 정책을 통해 "자신의 데이터만 읽고 쓸 수 있도록" 데이터베이스 레벨에서 엄격히 차단합니다.
2. **보안 강화 인증**: 구글 OAuth를 통해 회원가입 허들을 낮추고, 비밀번호 암호화 및 보관의 책임을 구글과 Supabase 측에 위임하여 높은 보안 수준을 유지합니다.
